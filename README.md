# Трекер личных финансов

Простое приложение командной строки для отслеживания личных финансов, написанное на Go.

## Описание

Это консольное приложение позволяет пользователям добавлять, просматривать и анализировать свои доходы и расходы. Оно разработано как учебный проект для начинающих разработчиков на Go, демонстрируя основные концепции языка, такие как структуры, методы, работа с файлами и CLI-взаимодействие.

## Функциональные требования

### 1. Добавление транзакций
- Пользователь может добавить транзакцию, указав:
  - Сумму (целое число)
  - Тип: "доход" или "расход"
  - Категорию (например, "еда", "транспорт", "зарплата", "подарки")
  - Опциональное описание
- Транзакции сохраняются в памяти во время сессии.

### 2. Просмотр транзакций
- Пользователь может вывести список всех добавленных транзакций.
- Каждая транзакция отображается с суммой, типом, категорией и описанием.

### 3. Анализ
- Приложение может вычислить и вывести:
  - Общий баланс (сумма доходов минус сумма расходов)
  - Общую сумму расходов
  - Общую сумму доходов

### 4. Интерфейс командной строки (CLI)
- Приложение должно отображать приветственное сообщение и меню с доступными командами.
- Пользователь вводит номер команды для выполнения действия.
- Поддерживаемые команды:
  1. Добавить транзакцию
  2. Показать все транзакции
  3. Показать статистику
  4. Выйти
- После выполнения команды (кроме выхода) меню отображается снова.

## Технические требования

### Язык и окружение
- Приложение написано на языке программирования Go (версия 1.24.1 или выше).
- Использует стандартную библиотеку Go.
- Не требует внешних зависимостей.

### Структура проекта
```
Personal_Finance_Tracker/
├── cmd/
│   └── tracker/
│       └── main.go
├── internal/
│   └── finance/
│       └── tracker.go
└── go.mod
```

- `cmd/tracker/main.go`: Точка входа в приложение. Содержит функцию `main()`, которая управляет потоком выполнения, отображает меню и обрабатывает ввод пользователя.
- `internal/finance/tracker.go`: Содержит основную логику приложения. Включает:
  - Структуру `Transaction` с полями: `Amount`, `Type`, `Category`, `Description`, `Date`.
  - Структуру `FinanceTracker` с полем `Transactions []Transaction`.
  - Методы для структуры `FinanceTracker`:
    - `AddTransaction(amount int, transactionType, category, description string) error`
    - `GetAllTransactions() []Transaction`
    - `GetBalance() int`
    - `GetTotalExpenses() int`
    - `GetTotalIncome() int`
- `go.mod`: Файл модуля Go.

### Реализация
- Использовать `time.Now()` для автоматического проставления даты у транзакций.
- Использовать `fmt` для ввода-вывода.
- Использовать `strings` и `strconv` для обработки ввода.
- Валидировать ввод пользователя (например, проверка, что сумма — это положительное число).
- Обрабатывать ошибки ввода и выводить понятные сообщения.

## Инструкции для разработчика

1. Создайте новый модуль Go в папке `Personal_Finance_Tracker`.
2. Создайте описанную структуру папок и файлов.
3. Начните с реализации структур и методов в `internal/finance/tracker.go`.
4. Затем реализуйте функцию `main` в `cmd/tracker/main.go`, которая будет использовать методы из `FinanceTracker`.
5. Тестируйте приложение, постепенно добавляя функциональность.
6. Обеспечьте хорошую обработку ошибок и читаемый пользовательский интерфейс.

Это ТЗ дает четкое направление для реализации проекта и включает в себя все необходимые детали для начинающего разработчика на Go.
